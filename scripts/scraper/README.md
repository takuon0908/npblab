# スクレイパー

npb.jpの順位表・試合日程・タイトルリーダーズページから日次でデータを収集し、Prisma DBにupsertする。

## 注意事項

npb.jp公式サイトのフッターには「掲載の情報、画像、映像等の二次利用および無断転載を固く禁じます」と明記されている。
事実データ（記録・結果）自体は著作権の保護対象になりにくいという整理はできるが、サイトの利用規約違反のリスクはゼロではない
（経緯のメモは Obsidian Vault の `04_Projects/野球を科学するサイト.md` を参照）。運用しながらリスクを許容できる範囲か随時見直すこと。

## 対象データと取得元

| データ | 取得元ページ | 書き込み先 |
|---|---|---|
| 順位表 | `npb.jp/bis/{year}/stats/std_{c\|p}.html` | `StandingsSnapshot` |
| 試合日程・結果 | `npb.jp/games/{year}/schedule_{MM}.html` | `Game` |
| タイトルリーダーズ（9部門×2リーグ） | `npb.jp/bis/{year}/stats/{lb\|lp}_{category}_{c\|p}.html` | `TitleLeaderSnapshot` |

- タイトルリーダーズは「順位・選手名(球団略称)・成績値」の3列のみで、打席数や投球回など詳細な個人成績は含まれない。
  独自指標（パークファクター調整成績など）に必要な個人の詳細成績は別ページの調査が必要（未着手）
- 選手には公式IDが取得できないため、`playerId`は`teamSlug-選手名`から生成した簡易スラッグ（同姓同名の選手がいる場合は衝突しうる。要改善）
- 球団名⇔略称のマッピングは`teams.ts`にハードコード（NPBの表記揺れ: 正式名称／1文字漢字略称の2種類）

## 実装状況

`parse.ts`（cheerioによるHTML解析）・`teams.ts`（球団マスタ）・`index.ts`（fetch→parse→Prisma upsert）まで実装済み。
実データでの動作確認済み（2026年7月時点、順位表12件・試合134件・リーダーズ1331件を正常に取得）。

未実装:
- 複数月にまたがる残り試合の取得（現在は当月のみ）
- 個人の詳細成績（打席数・投球回など）ページの特定と取得

## 実行方法

```
npm run scrape
```
